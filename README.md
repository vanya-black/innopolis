# Эмбеддинги и точка: как нейросети понимают смысл

Материалы лекции про эмбеддинги: **слайды** (Marp) + **два живых демо** на Streamlit.
Одна идея через всю лекцию — *что угодно превратить в вектор и найти ближайшее по смыслу*:
текст и картинки кодируются в точки одного пространства, а «похожее» — это просто
ближайшие точки.

```
lecture/             слайды (Marp): .md
game_map_demo/       демо №1 — карта игр (текстовые эмбеддинги)
sketch_search_demo/  демо №2 — нарисуй и найди похожее (CLIP)
```

## Лекция

`lecture/lecture-slides.md` — исходник слайдов в [Marp](https://marp.app/).
Рядом все картинки.

```bash
# пересобрать слайды (нужен marp-cli)
marp lecture/lecture-slides.md -o lecture/lecture-slides.html   # или --pdf
```

Канва рассказа: как компьютер видит текст и картинки → векторы как «карта смыслов»
→ что с векторами делать (похожесть = близость) → бизнес-применения → связь с
ChatGPT/Claude (токены, авторегрессия) → **два демо вживую**.

## Демо

| Демо | Что показывает | Модель | Запрос → ответ |
|------|----------------|--------|----------------|
| [`game_map_demo/`](game_map_demo/README.md) — 🎮 **Карта игр** | TOP-1000 игр как точки на 2D-карте; поиск соседей по игре или свободному тексту (`open world rpg with dragons`). Жанры сами собираются в «материки» | `all-MiniLM-L6-v2` (текст, 384D) | фраза → вектор → ближайшие игры |
| [`sketch_search_demo/`](sketch_search_demo/README.md) — ✏️ **Нарисуй — найди похожее** | Рисунок на холсте → визуально похожие фотографии из галереи Sketchy (12 500 фото) | CLIP `clip-ViT-B-32` (картинки, 512D) | рисунок → вектор → ближайшие фото |

Оба демо устроены одинаково (`collect.py` собирает корпус → `embed.py` заранее
считает эмбеддинги в `artifacts/` → `app.py` грузит готовый индекс) и делят один
venv — `game_map_demo/.venv`. Тяжёлые эмбеддинги посчитаны заранее, поэтому в
реальном времени остаются лишь вектор запроса и поиск ближайших.

**Запуск и все детали — в README каждого демо.**

## Данные, артефакты и ключи

- **Артефакты** (`*/artifacts/`, лёгкие `.npy`/`.json`) лежат в git — этого хватает
  для дашбордов.
- **Фотографии Sketchy** (~200 МБ) в git **не** хранятся: их детерминированно
  восстанавливает `sketch_search_demo/collect.py` (индекс остаётся выровненным).
- **RAWG API-ключ** нужен только для пере-сбора игр (`game_map_demo/collect.py`).
  Скопируйте `.env.example` → `.env` и впишите ключ ([rawg.io/apidocs](https://rawg.io/apidocs)).
  `.env` в git не попадает; дашбордам ключ не нужен.
