# 🎮 Карта игр в пространстве эмбеддингов

Демо к лекции про эмбеддинги. Выгружаем TOP-1000 игр, превращаем каждую в вектор
(«координату смысла»), раскладываем на 2D-карту и ищем ближайших соседей — по
выбранной игре или по свободному текстовому запросу.

Три файла = три шага:

| Шаг | Файл | Что делает |
|-----|------|------------|
| 1. Сбор данных | `collect.py` | тянет TOP-N игр из RAWG (имя, жанр, теги, описание) → `artifacts/games.json` |
| 2. Эмбеддинги + карта | `embed.py` | кодирует игры моделью **all-MiniLM-L6-v2**, строит 2D-проекции (UMAP, PCA) → `artifacts/` |
| 3. Дашборд | `app.py` | карта + поиск соседей (Plotly + Streamlit), косинусная близость |

`embed.py` ещё и общий «движок»: дашборд берёт оттуда модель, эмбеддинг запроса и
поиск соседей — весь смысловой код в одном месте.

## Запуск

```bash
cd game_map_demo
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python collect.py     # 1) выгрузка игр (нужен RAWG-ключ, см. ниже)
python embed.py       # 2) эмбеддинги + 2D-проекции
streamlit run app.py  # 3) дашборд на http://localhost:8501
```

…или одной командой: `bash run_all.sh`.

Готовые артефакты уже лежат в `artifacts/` — для дашборда шаги 1–2 можно пропустить.

### RAWG-ключ (только для шага 1)

Бесплатный ключ на [rawg.io/apidocs](https://rawg.io/apidocs). Положите в `.env`
(в каталоге демо или в корне репозитория):

```
RAWG_API_KEY=ваш_ключ
```

## Как это работает (для слайда)

1. `embed.py` заранее перевёл каждую из 1000 игр в вектор из 384 чисел.
2. Запрос (название игры или фраза) проходит через **ту же** модель → **то же** пространство.
3. Ищем игры с ближайшими векторами (косинусная близость) — это и есть «похожие».

Отвечает мгновенно: тяжёлые эмбеддинги посчитаны заранее, в реальном времени —
только вектор запроса и поиск ближайших.

## Подсказки

- Свободный текст: `relaxing farming simulation`, `souls-like with hard bosses`,
  `open world fantasy rpg with dragons` — карта подсветит ближайшие игры.
- Точки рядом ⇒ игры похожи по смыслу; цвет = жанр; жанры сами собираются в «материки».
- PCA — линейная проекция (общая форма), UMAP — нелинейная (плотнее кластеры).
